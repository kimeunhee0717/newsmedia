- 대화형 심화 분석과 문서화: **ChatGPT Search**

## 3. 도구별 상세 분석

## A. Perplexity

### 강점

- 출처 링크를 빠르게 확인하기 좋음
- “짧은 시간에 신뢰 가능한 후보 자료”를 추리기 좋음
- 탐색 초기 단계 속도가 빠름

### 약점

- 긴 맥락 분석/문서화는 별도 도구 보완이 필요
- 결과를 보고서 템플릿으로 다듬는 후작업이 남음

### 추천 업무

- 시장 조사 시작점 탐색
- 수치/근거 링크 빠른 수집

---

## B. ChatGPT Search

### 강점

- 심화 질문을 이어가며 논리 구조를 다듬기 좋음
- 리서치 결과를 목차/표/액션 형태로 재구성하기 쉬움
- 문서 초안까지 바로 연결 가능

### 약점

- 탐색 초기 속도는 Perplexity보다 느리게 느껴질 때가 있음
- 출처 검증은 사용자가 한 번 더 체크하는 습관이 필요

### 추천 업무

- 리서치 결과 보고서화
- 의사결정 문서/브리핑 노트 작성

---

## C. Gemini

### 강점

- 구글 문맥(검색, 문서, 워크스페이스) 연결 시 사용감이 좋음
- 멀티모달 기반 입력 활용이 자연스러움

### 약점

- 특정 주제에서는 답변 밀도를 끌어올리기 위해 재질문 설계가 필요
- 결과를 팀 표준 문서로 정리할 때 후편집이 필요할 수 있음

### 추천 업무

- 구글 환경 중심의 협업팀
- 문서 + 웹 탐색 혼합 작업

## 4. 실무용 운영 패턴(추천)

## 패턴 1: 빠른 리서치

1. Perplexity로 1차 근거 링크 수집  
2. ChatGPT Search로 핵심 요약/쟁점 정리  
3. 최종 문서화

## 패턴 2: 보고서 작성

1. ChatGPT Search로 목차/가설 생성  
2. Perplexity로 근거 보강  
3. Gemini로 추가 반례 탐색

## 패턴 3: 팀 브리핑

1. 도구 2종 이상으로 교차 검증  
2. 공통 결론/상충 포인트 분리  
3. “확인 필요” 태그로 리스크 명시

---

## 5. 바로 쓰는 검색 프롬프트

```text
주제: [주제 입력]
목표: 실무 의사결정에 필요한 근거 수집

출력 형식:
1) 핵심 결론 5줄
2) 근거 출처 8개(링크 + 요약 + 발행일)
3) 상충되는 주장 3개
4) 추가 확인 필요 항목 5개

규칙:
- 추측 금지
- 출처 없는 단정 금지
- 최신성 이슈는 날짜 명시
```

---

## 6. 실패를 줄이는 체크리스트

1. 출처 날짜를 반드시 확인한다  
2. 1개 도구 결과만으로 결론 내리지 않는다  
3. 수치/정책 문장은 원문 링크를 같이 남긴다  
4. 반대 근거를 의도적으로 찾는다  
5. 최종 문서에 “확인 필요” 항목을 남긴다

## 7. 최종 정리

AI 검색은 “답변 잘 받기”보다 “검증 가능한 근거 체계 만들기”가 핵심입니다.

- 탐색 시작: Perplexity
- 분석 정리: ChatGPT Search
- 보강 점검: Gemini

이 순서로 운영하면 리서치 품질과 속도를 동시에 확보하기 좋습니다.
