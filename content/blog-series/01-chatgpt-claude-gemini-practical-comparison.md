## 0. 먼저 결론부터

이 글의 결론은 단순합니다.

- 범용성, 속도, 후속 편집 반영: **ChatGPT**
- 긴 문서 맥락 유지, 안정적인 문체: **Claude**
- 검색성 작업, 구글 연동 문맥: **Gemini**

실무에서는 “하나만 고르는 방식”보다 “역할 분담”이 훨씬 좋았습니다.

- 초안/아이디어: ChatGPT
- 정돈/문체 통일: Claude
- 사실 보강/검색성 확인: Gemini

## 3. 과제별 상세 비교

## A. 블로그 글쓰기(초안 작성)

### 공통 프롬프트

```text
주제: "AI 도구를 업무에 도입할 때 생기는 실수"
대상: 실무자(비개발자 포함)
형식: H2/H3 구조, 1200~1500자, 체크리스트 포함
톤: 단정하고 실용적, 과장 금지
```

### ChatGPT 결과 특성

- 장점
  - 목차 구조를 빠르게 잡고, 실행 문장으로 이어주는 능력이 좋음
  - “더 짧게”, “더 실무형으로” 같은 후속 지시 반영이 빠름
- 아쉬운 점
  - 프롬프트가 짧으면 문장이 일반론으로 퍼질 수 있음

### Claude 결과 특성

- 장점
  - 문장 연결이 자연스럽고 문체 안정성이 높음
  - 긴 단락에서도 논리 연결이 무너지지 않음
- 아쉬운 점
  - 제목/후킹 문구는 다소 보수적으로 나올 때가 있음

### Gemini 결과 특성

- 장점
  - 검색형/브리핑형 문서 톤에서 강점
  - 구글 문맥과 맞물릴 때 흐름이 좋음
- 아쉬운 점
  - 밀도 높은 글을 만들려면 재지시가 더 필요함

정리:

- “빠르게 게시 가능한 초안”은 ChatGPT 우세
- “문장 완성도”는 Claude 우세

---

## B. 긴 문서 요약

### 공통 프롬프트

```text
아래 문서를 1) 핵심 5줄, 2) 실행 체크리스트 7개, 3) 리스크 3개로 요약해줘.
모르는 내용은 추측하지 말고 "정보 부족"으로 표시해줘.
```

### ChatGPT

- 핵심 포인트 추출 속도 빠름
- 요약 결과를 표/체크리스트로 재구성하기 쉬움

### Claude

- 긴 문맥 유지가 안정적
- “왜 중요한지”를 덧붙여 설명하는 품질이 좋음

### Gemini

- 웹 문맥을 섞은 요약이 필요할 때 유용
- 다만 깊이 있는 요약은 프롬프트 설계가 중요

정리:

- 보고서형 요약: Claude
- 빠른 실무 요약: ChatGPT

---

## C. 코딩 보조(생성/디버깅)

### 공통 프롬프트

```text
다음 Next.js 빌드 에러 로그를 읽고:
1) 원인 후보 3개
2) 재현 확인 방법
3) 최소 수정 패치
를 순서대로 제시해줘.
```

### ChatGPT

- 문제 원인 분해가 빠르고, 패치 제안이 실용적
- “왜 이 수정이 맞는지” 설명이 명확함

### Claude

- 긴 코드 문맥 리뷰가 안정적
- 리뷰 스타일 피드백이 깔끔함

### Gemini

- 짧은 스니펫/간단 오류 대응에는 충분
- 복합 디버깅은 재질문 설계가 필요

정리:

- 실무 코딩 보조 기본값: ChatGPT
- 대형 코드/문서 리뷰: Claude

- 스펙 문서 리뷰: Claude
- 검색성 참고 수집: Gemini

## 5. 실패를 줄이는 프롬프트 템플릿

아래 템플릿을 쓰면 결과 품질이 안정됩니다.

```text
[역할]
당신은 실무 문서 편집자다.

[목표]
아래 초안을 '실행 가능한 가이드'로 재작성한다.

[제약]
- 과장 금지
- 모르는 사실 추정 금지
- 불명확하면 "확인 필요" 표시
- 1200~1500자

[출력 형식]
1. 요약
2. 핵심 포인트 5개
3. 바로 실행할 체크리스트 7개
```

핵심은 “역할 + 목표 + 제약 + 출력형식”을 함께 주는 것입니다.

## 6. 자주 하는 실수 5가지

1. 도구 하나로 모든 업무를 해결하려고 함  
2. 프롬프트를 너무 짧게 써서 결과가 흔들림  
3. 1회 결과만 보고 성능을 단정함  
4. 사실 검증 없이 게시함  
5. 후속 편집 지시를 안 하고 “첫 결과”만 사용함
